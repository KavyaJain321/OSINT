Build Deduplication Checker 
Two-layer check — URL uniqueness and content fingerprint 
CURSOR 
PROMPT 
CURSOR PROMPT 
Build src/services/dedup-checker.js 
  
generateContentHash(content) → string 
- Take first 600 chars of content 
- Normalize: lowercase, remove extra whitespace and punctuation 
- Return MD5 hex string using crypto (Node built-in) 
  
isDuplicate(url, content, clientId) → Promise<{isDuplicate, reason}> 
- Layer 1: Query articles WHERE url = url AND client_id = clientId 
  → if found: return { isDuplicate: true, reason: "url_exists" } 
- Layer 2: Generate hash, query WHERE content_hash = hash AND client_id 
  → if found: return { isDuplicate: true, reason: "content_exists" } 
- If neither: return { isDuplicate: false, reason: null } 
- On DB error: log warning but return isDuplicate: false 
  (better to store potential duplicate than miss real article) 
  
normalizeUrl(url) → string 
- Remove trailing slashes 
- Remove UTM parameters (?utm_source, ?ref, etc.) 
- Lowercase the domain portion 
- Call this on every URL before dedup check

Create a PostgreSQL function that:

4. Returns the modified event 
  
The function signature for Supabase hook: 
CREATE OR REPLACE FUNCTION public.custom_access_token_hook(event jsonb) 
RETURNS jsonb LANGUAGE plpgsql AS $$ 
DECLARE 
  claims jsonb; 
  user_role text; 
  user_client_id uuid; 
BEGIN 
  SELECT role, client_id INTO user_role, user_client_id 
  FROM public.users WHERE id = (event->>user_id)::uuid; 
  -- Add to claims and return modified event 
END; $$;

## Non-Negotiable Rules 
1. Every database query MUST filter by client_id (unless SUPER_ADMIN)

4. Always use ES modules (import/export) — never require()

7. Wrap every async route handler in try/catch

- Descriptive console.log with prefix: [Scraper], [AI], [Chat], [Auth] 
- Route handlers thin — business logic in services/ or ai/ files

## Database Patterns 
- Always use supabase from src/lib/supabase.js (singleton) 
- const { data, error } = await supabase.from(...) — always check error 
- Use .limit() on all queries — prevent accidental full-table scans 
  
## Scraper Rules 
- Never scrape more than 50 articles per source per run 
- Always close Playwright browsers in finally blocks 
- Never run more than 2 Playwright browsers simultaneously 
- Always update last_scraped_at after each attempt (success or fail)
